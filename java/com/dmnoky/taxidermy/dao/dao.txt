1.	в файле appconfig-data.xml описываем BasicDataSource, который хранит имя драйвера для соединения с БД – JDBC Driver, путь к БД, логин и пароль;
2.	там же, описываем LocalSessionFactoryBean для корректной работы фреймворка Hibernate – указываем ссылку на BasicDataSource, пакет со всеми классами – моделями/сущностями, хранимыми в БД, и прочие настройки.
3.	объявляем аннотацию @Transactional видимой приложению. У hibernate появляется возможность работать с транзакциями – в случае выполнения COMMIT, все изменения, выполненные в рамках одной транзакции, сохранятся в физическую БД, иначе ROLL BACK, происходит откат всех изменений до места, помеченного аннотацией @Transactional. Данная аннотация используется в @Service слое, который позволяет создавать транзакционные операции с использованием нескольких DAO модулей (классы, содержащие CRUD методы для конкретных сущностей). Так, @Service слой определяет бизнес логику, для реализации которой он обращается к разным DAO. (ПРИЛОЖЕНИЕ 5.5, 6)
4.	связываем все DAO – классами, работающими напрямую с БД через SQL – JDBC код, с LocalSessionFactoryBean, который уже хранит все надстройки БД. (ПРИЛОЖЕНИЕ 5.5, 7)
