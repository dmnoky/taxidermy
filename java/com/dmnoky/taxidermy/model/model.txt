1.	Animal – основная сущность, определена как главный товар/заказ. 
2.	AnimalListCountTemplate – класс-обертка для более удобного хранения листа Animal и размера всего семейства Animal, определенного типа, хранимого в БД. 
3.	Subsidiary – второстепенная сущность, определяющая семейство Animal.
4.	Type – второстепенная сущность, определяющая тип Animal.
5.	Address – модель, дополняющая своими свойствами другие сущности. Определяет адрес соответствующих объектов.
6.	Product – основная сущность, определена как главный товар/заказ.
7.	User – модель, являющаяся родителем для следующих классов – Client и Worker.
8.	Client – сущность, расширяющая класс User. Определяет клиента.
9.	Worker – сущность, расширяющая класс User. Определяет работника.
10.	 AnimalCount – класс, который хранит ссылки для связи количества приобретенного товара Animal, используется при POST запросе.
